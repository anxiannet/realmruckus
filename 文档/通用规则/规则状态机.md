# 通用规则状态机

规则版本：V1.3  
更新时间：2026-07-14

## 一、目的

本文件将规则表示为可实现、可测试的状态转换，供网站、数字原型、AI模拟器和自动测试共同使用。

## 二、核心状态

```text
GameState
├── players
├── current_player
├── turn_number
├── public_deck
├── discard_pile
├── area_deck
├── center_areas
├── controlled_areas
├── hands
├── pending_effects
├── no_progress_round
└── winner
```

## 三、回合状态

```text
TURN_START
→ DRAW
→ CHOOSE_ACTION
→ RESOLVE_ACTION
→ RESOLVE_FORCED_EFFECTS
→ RECYCLE_EMPTY_AREAS
→ CHECK_IMMEDIATE_WIN
→ HAND_LIMIT
→ CHECK_DECK_END
→ TURN_END
```

任何强制效果产生新的空区域时，必须先完成区域回流，再继续下一结算步骤。

## 四、主要行动状态

### 部署

```text
SELECT_UNIT
→ SELECT_LEGAL_AREA
→ MOVE_CARD_TO_AREA
→ UPDATE_CONTROL
→ CHECK_IMMEDIATE_WIN
→ RESOLVE_ENTER_EFFECT
→ RECYCLE_EMPTY_AREAS
→ CHECK_IMMEDIATE_WIN
```

### 进攻

```text
SELECT_ORIGIN
→ SELECT_TARGET
→ SELECT_ATTACKERS
→ ORDER_ATTACKERS
→ RESOLVE_DUELS
→ RESOLVE_ATTACK_RESULT
→ RECYCLE_EMPTY_AREAS
→ RESTORE_SURVIVORS
→ CHECK_IMMEDIATE_WIN
```

### 效果牌

```text
SELECT_EFFECT_CARD
→ VALIDATE_COST_AND_TARGETS
→ PAY_COST
→ OPEN_RESPONSE_WINDOW
→ RESOLVE_OR_CANCEL
→ MOVE_CARD_TO_DESTINATION
→ RECYCLE_EMPTY_AREAS
→ CHECK_IMMEDIATE_WIN
```

## 五、战斗状态

单次对决：

```text
attacker_damage += defender.level
defender_damage += attacker.level
```

同时计算，不因任一方本次被击败而取消另一方伤害。

进攻结束条件：

- 防守方无存活单位；或
- 所有进攻单位均已完成一次攻击。

成功条件：

```text
defender_survivors == 0 and attacker_survivors >= 1
```

双方全灭不构成进攻成功。

## 六、区域回流状态

```text
IF controlled_area.units == 0
THEN
  remove control
  clear temporary protection
  move area to area_deck
  shuffle area_deck
  refill center to configured size
  increment area_return_count
```

回流检查必须循环执行，直到没有空的已控制区域。

## 七、即时胜利

任何控制权变化后执行：

```text
count controlled areas by group
if any group_count >= 3:
    set winner immediately
```

即时胜利高于后续非必要步骤，不等待回合结束。

## 八、牌库耗尽状态

公共牌库首次为空后记录 `deck_empty_once = true`。

每个回合记录：

```text
played_public_card_this_turn
control_changed_this_turn
```

完整一轮所有玩家均未打牌且无控制权变化时，进入最终比较。

## 九、非法状态

实现必须阻止：

- 同一区域出现多个所有者的驻军；
- 超过驻军上限；
- 从手牌直接发起进攻；
- 攻击中央无主区域；
- 攻击自己的区域；
- 使用没有合法目标且无法结算的行动；
- 已回流区域仍保留所有者或临时保护；
- 被击败单位仍留在区域；
- 胜利已确定后继续改变必要游戏状态。
