# 标准组件模型

规则版本：V1.3  
更新时间：2026-07-14

## 一、公共牌库模型

标准公共牌库共40张：

| 类型 | 等级 | 数量 |
|---|---:|---:|
| 单位 | 1 | 10 |
| 单位 | 2 | 8 |
| 单位 | 3 | 10 |
| 单位 | 4 | 6 |
| 单位 | 5 | 2 |
| 效果牌 | 无 | 4 |
| 合计 |  | 40 |

本表只定义数量与等级分布，不定义任何具体名称、角色、效果文本或世界观表达。

## 二、区域牌库模型

```text
区域总数：12
分组数量：4
每组区域：3
每张区域牌：1份
```

通用标识建议：

```text
GROUP-01 / AREA-01
GROUP-01 / AREA-02
GROUP-01 / AREA-03
...
GROUP-04 / AREA-03
```

具体表达可将分组与区域映射为任意主题名称，但不得改变3×4结构。

## 三、机制身份模型

每个唯一机制卡应具有稳定的机制身份，不随皮肤、语言或插画变化。

推荐字段：

```text
mechanic_id
card_kind
level
copy_count
effect_id
rules_version
status
```

具体表达卡面推荐字段：

```text
expression_id
skin_id
language
card_code
mechanic_id
name
art_version
text_version
layout_version
status
```

## 四、卡号与版本

- `mechanic_id`：通用机制身份，不印在卡面。
- `card_code`：具体表达的实际卡面版本号。
- 卡面完全相同的复制卡共用同一 `card_code`。
- 名称、语言、插画、文本、等级、图标、边框或版式变化时，分配新 `card_code`。
- 旧卡号永久保留，不复用。

## 五、效果分类建议

通用层只维护效果类型与参数，不维护主题名称。

示例分类：

```text
RETURN_OWN_UNIT
DRAW_THEN_DISCARD
VIEW_RANDOM_HAND_CARD
SOCIAL_CALL_EXCHANGE
SOCIAL_CALL_DRAW
RECOVER_LOW_LEVEL_UNIT
DEFEAT_UNIT_BY_LEVEL
RETURN_UNIT_BY_LEVEL
LOOK_TOP_AND_CHOOSE
TAKE_RANDOM_HAND_CARD
SECRET_SWAP
TAKE_UNIT_TO_HAND
TAKE_UNIT_TO_AREA
RETURN_MULTIPLE_UNITS
PLAY_EXTRA_UNIT
IMMEDIATE_ATTACK
CANCEL_EFFECT
PROTECT_AREA
```

每个效果必须另行维护：

- 触发时机
- 目标类型
- 数量
- 等级限制
- 所有权变化
- 是否可选
- 无合法目标时处理
- 与区域回流的关系

## 六、扩展兼容

扩展可以增加新的机制身份或增加既有机制的复制数量，但必须：

- 明确基础牌库与合并牌库数量；
- 保持通用核心规则兼容；
- 单独进行2、3、4人回归测试；
- 检查牌库耗尽率、平均回合和区域回流；
- 不用主题名称定义通用逻辑。
