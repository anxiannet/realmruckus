# 通用战斗方案评分模型

版本：GEN-BATTLE-SCORE-001  
生效日期：2026-07-14

## 一、文件定位

本文件定义AI对一项具体进攻方案的局面价值评分。

它用于：

- 比较不同出兵区域；
- 比较不同目标区域；
- 比较不同进攻单位组合；
- 比较进攻顺序与防守目标顺序；
- 决定普通AI是否执行该次进攻；
- 为压力模型提供更低的执行阈值。

本模型不评价卡牌的理论技能强度，也不负责手牌保留和弃牌排序。

## 二、输入

```text
player
origin_area
target_area
battle_result
mechanism_priority
```

`battle_result` 至少包含：

```text
success
attack_order
defeated_attackers
defeated_defenders
surviving_attackers
surviving_defenders
```

## 三、单位价值

源模型对被击败单位使用：

```text
unit_value
= level × 5
+ ai_base_priority / 20
```

因此：

```text
value_defeated
= Σ unit_value(defeated_defenders)

value_lost
= Σ unit_value(defeated_attackers)
```

初始交换分：

```text
exchange_score
= value_defeated - value_lost
```

说明：

- `ai_base_priority` 是AI卡牌优先级，不是技能强度评分；
- 该项只为同等级单位提供细微价值差；
- 等级仍是基础战斗价值的主要部分。

## 四、分组进度与威胁

```text
own_same_group
= 进攻玩家在目标分组中已控制的区域数量

target_owner_same_group
= 目标所有者在目标分组中已控制的区域数量
```

`own_same_group` 用于衡量夺取目标后接近胜利的程度。

`target_owner_same_group` 用于衡量此次进攻是否能阻止对手完成同组胜利。

## 五、正式human_like基线

当前迁移的收紧版正式基线：

```text
score = value_defeated - value_lost
```

若进攻成功：

```text
score += 60
score += own_same_group × 40

if own_same_group >= 2:
    score += 220

if target_owner_same_group >= 2:
    score += 100
```

若进攻失败：

```text
score -= 22

if defeated_defenders is empty:
    score -= 24

if surviving_attackers is empty:
    score -= 18

if defeated_defenders exists and value_defeated > value_lost:
    score += 8

if both sides have no survivors:
    score += 12
```

若全部驻军出兵且未占领目标区域：

```text
score -= 20
```

## 六、历史基线

原模拟器较早版本使用：

```text
成功夺区：+55
同组进度：own_same_group × 35
完成同组胜利前置：+200
阻止对手胜利前置：+90
失败：-8
失败但清空防守方：+20
全部出兵且无存活进攻单位：-25
```

当前通用迁移采用收紧后的正式基线：

```text
60 / 40 / 220 / 100
失败惩罚22
无击败防守方额外惩罚24
无进攻方存活额外惩罚18
清空出兵区失败额外惩罚20
```

历史参数保留用于回归比较，不作为当前默认模型。

## 七、进攻执行阈值

收紧版源模型：

| AI模式 | 普通进攻最低分 |
|---|---:|
| pressure | 8 |
| human_like | 14 |
| forced_attack | -999 |

含义：

- `human_like` 只执行明显正收益的进攻；
- `pressure` 允许执行收益较低但仍略有价值的进攻；
- 强制进攻效果必须执行合法的最佳方案，因此使用极低阈值。

历史旧模型曾使用：

```text
pressure = -5
human_like = 10
```

该组参数只作为历史对照。

## 八、方案生成与选择

必须枚举：

1. 所有合法出兵区域；
2. 所有合法目标区域；
3. 1至驻军上限数量的进攻单位组合；
4. 所有进攻单位顺序；
5. 所有合法防守目标顺序；
6. 每个方案的战斗结果。

选择：

```text
best_plan = max(all_legal_plans, key=battle_score)
```

如果：

```text
best_plan.score < ai_threshold
```

则普通AI不进攻。

## 九、评分解释

### 成功夺区

成功奖励远高于普通单位交换，原因是区域控制直接关联胜利条件。

### 同组进度

`own_same_group × 40` 表示越接近同组3区胜利，目标区域越有价值。

### 立即胜利奖励

当进攻前已经控制目标分组2个区域时，成功夺区将立即获胜，因此额外增加高额奖励。

### 阻止对手奖励

目标所有者已经控制同组2区时，夺取其中一个区域可阻止其立即胜利，因此增加高额奖励。

### 失败惩罚

失败攻击会消耗主要行动并暴露出兵区域，必须产生明确交换价值才值得执行。

### 双方全灭

双方全灭会使目标区域回流，可能破坏对手的分组进度，因此保留少量正价值，但不视为进攻成功。

### 清空出兵区风险

全部驻军出兵但未占领目标时，出兵区可能失去全部单位并回流，因此额外扣分。

## 十、后续可扩展字段

后续引擎开发可以增加，但不得在迁移阶段直接替换当前基线：

```text
origin_area_win_risk
origin_area_protection_value
target_area_protection_value
hand_followup_value
known_counter_risk
area_deck_information
position_bias
```

任何新增字段必须通过模拟和实体试玩校准。

## 十一、与其他评分系统的边界

```text
战斗方案评分
≠ 技能强度评分
≠ AI卡牌优先级评分
```

- 技能强度评分评价规则效果本身；
- AI卡牌优先级评价卡牌保留与使用顺序；
- 战斗方案评分评价当前局面下的一次具体进攻。
