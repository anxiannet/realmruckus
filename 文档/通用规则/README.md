# 通用规则层

规则版本：V1.3  
更新时间：2026-07-14

## 文件定位

本目录是项目通用机制、规则、卡表、评分模型、数学模型、测试规范和验证基线的唯一来源。

本层不包含：

- 具体卡名
- 角色名
- 世界观名称
- 阵营名
- 地名
- 皮肤专属术语
- 具体品牌卡号
- 包装或美术表达

多个具体产品表达均通过映射引用本通用规则层。

## 主从关系

```text
通用规则层
├── 具体表达A
├── 具体表达B
└── 后续具体表达
```

具体表达只能替换：

- 名称
- 世界观
- 插画
- 语言
- UI
- 包装
- 卡面编号前缀

具体表达不得自行修改：

- 核心胜利条件
- 回合结构
- 战斗结算
- 区域回流
- 手牌上限
- 驻军上限
- 牌库耗尽结算
- 已锁定的组件模型
- 通用机制参数
- 通用评分公式

## 唯一数据源

```text
data/core_set.json
data/expansion_1.json
```

Markdown卡表只作人类可读说明，若与JSON冲突，以JSON为准。

## 维护原则

- 通用机制只在本层维护。
- 具体表达通过映射引用本层，不复制整套核心规则或通用卡表。
- 具体表达专属内容不得反向写入通用层。
- 三套评分系统必须独立维护，不得合并。
- 通用规则变化必须同步检查卡表、测试模型和历史基线。
- 每个通用文件应记录来源、抽取范围和排除的具体表达。
