# 通用内容迁移范围

生效日期：2026-07-14

## 一、迁移目标

从原项目中只抽取可被多个具体产品共同使用的通用内容。

新主仓库负责维护：

- 通用核心机制
- 通用规则流程
- 通用标准版卡表
- 通用扩展包卡表
- 通用数学模型
- 通用测试模型
- 通用技能强度评分方法
- 通用AI决策模型
- 通用战斗评分模型
- 通用生产与审核流程

《夕妖：抢地盘》和 Realm Ruckus 均为这些通用内容的具体表达。

## 二、禁止迁移的具体表达

以下内容不得迁入通用层：

- 具体卡名
- 角色名
- 怪物名
- 法宝名
- 地名
- 阵营名
- 世界观
- 剧情
- 西游专属术语
- Realm Ruckus专属术语
- QDP或RRK具体卡号
- 中文或英文皮肤文案
- 具体插画
- 具体包装
- 具体Logo
- 具体产品专属UI
- 以具体名称为键的代码和统计结果

如源文件混合通用机制与具体表达，只抽取通用部分，不能原样复制整个文件。

## 三、规则迁移原则

通用规则使用抽象术语：

```text
Card
Unit Card
Effect Card
Area
Group
Player
Owner
Level
Deploy
Attack
Return
Defeat
Control
```

不得在通用规则中出现任何具体产品名称或专属名词。

## 四、卡表迁移原则

通用卡表只维护：

- `mechanic_id`
- `kind`
- `level`
- `copies`
- `effect_id`
- `timing`
- `parameters`

具体表达只通过映射维护：

- 本地化名称
- 本地化文案
- 插画
- 卡面编号
- 语言
- 排版

## 五、测试代码迁移原则

测试逻辑可以迁移，具体表达代码不能直接迁移。

允许迁移：

- 对局状态结构
- 战斗枚举算法
- AI决策公式
- 批量模拟流程
- 随机种子策略
- 统计指标
- 回归比较方法
- 压力测试方法

禁止直接迁移：

- 通过具体卡名判断效果的分支
- 通过具体地名判断分组的代码
- 以具体角色名记录逐卡统计的结构
- 具体产品专属输出文案

迁移方法：

```text
读取源代码
→ 识别通用算法
→ 记录算法来源
→ 生成无具体表达的通用文件
→ 保留行为说明与测试依据
```

## 六、测试结果迁移原则

只迁移可脱离具体表达解释的统计结果：

- 玩家人数
- 牌库数量
- 区域数量
- 中央区域数量
- AI模型类型
- 总局数
- 随机种子
- 平均回合
- 中位回合
- P90
- 直接胜利率
- 最终结算率
- 超时率
- 牌库耗尽率
- 成功进攻次数
- 失败进攻次数
- 区域回流次数

不迁移：

- 具体卡名的触发次数
- 具体角色使用后胜率
- 具体皮肤卡牌统计

如需要保留逐机制统计，必须先转换为 `mechanic_id` 或 `effect_id`。

## 七、技能评分系统迁移原则

迁移通用方法，不迁移具体技能名称和具体卡牌评分表。

允许迁移：

- 无上限累计整数分
- 锚点优先原则
- 受影响战力修正
- 数量修正
- 持续时间修正
- 区域连锁修正
- 行动节奏修正
- 资源差修正
- 选择自由度修正
- 多玩家与多区域修正
- 星级或等级限制扣分
- 前置条件扣分
- 使用成本扣分
- 反制风险扣分
- 强度区间定义

禁止迁移：

- 以具体卡名命名的评分锚点
- 具体角色技能评分
- 具体扩展包卡牌评分
- 世界观符合度评分

具体卡名锚点必须转写为抽象机制锚点，例如：

```text
返回一个区域内全部单位并使区域回流
击败一个区域内全部单位并使区域回流
获得一个区域内全部单位到手牌
额外部署一个单位
立即进行一次进攻
取消一次效果牌结算
```

## 八、三套评分系统边界

必须分别迁移：

```text
技能强度评分
AI卡牌优先级评分
具体战斗方案评分
```

三者不得合并。

- 技能强度评分：评价规则效果的理论强度。
- AI卡牌优先级：帮助AI选择部署、弃牌、保留和使用顺序。
- 战斗方案评分：评价当前局面下某次具体进攻方案。

所有公式必须移除具体名称依赖。

## 九、来源记录

通用文件应记录：

- 原仓库
- 原路径
- 原文件SHA
- 相关提交
- 抽取范围
- 被排除的具体表达
- 生成的通用文件路径

来源记录用于追溯，不代表复制原文件。

## 十、迁移完成标准

只有满足以下条件才算完成：

1. 所有通用机制已抽取。
2. 所有通用规则流程已抽取。
3. 通用标准版与扩展包数据已建立。
4. 通用数学模型已建立。
5. 通用测试算法与统计结构已抽取。
6. 通用技能强度评分系统已建立。
7. AI优先级与战斗评分系统已分别抽取。
8. 新仓库通用文件中不存在具体卡名和具体世界观表达。
9. 测试结果已转换为通用参数统计。
10. 完成通用内容迁移审核。

迁移审核通过后，再继续开发完整通用引擎。
