# 通用版本基线裁决记录

裁决日期：2026-07-14

## 一、冲突背景

原项目历史文件中同时出现过：

```text
42张标准版 + 18张扩展 = 60张
```

以及当前正式卡表和模拟器采用的：

```text
40张标准版 + 10张扩展 = 50张
```

为避免新主仓库继续并存两套标准，本文件做出正式裁决。

## 二、当前通用基线

通用规则V1.3当前基线确定为：

```text
标准版：40张公共牌
第一扩展包：10张公共牌
合并牌库：50张
区域牌：12张
```

对应数据版本：

```text
GEN-CORE-001
data/core_set.json

GEN-EXP-001
data/expansion_1.json
```

## 三、裁决依据

采用40+10方案的依据：

1. 原仓库当前 `规则/V1.3-基础卡表.md` 为40张结构。
2. 原仓库当前第一扩展包卡表明确为10张。
3. 原仓库正式批量模拟器支持40张标准版和50张合并牌库。
4. 已迁移的最新3000局历史基线使用40张与50张口径。
5. 42张与60张数据出现在较早候选状态与历史测试中，不对应当前正式卡表。

## 四、历史版本定位

```text
42张标准版：历史候选
60张合并牌库：历史候选
```

历史数据可以保留用于比较，但不得作为当前组件数量、生产清单或默认模拟配置。

## 五、变更边界

未来如需恢复42张、60张或采用其他数量，必须：

1. 新建规则与卡表版本；
2. 修改JSON唯一数据源；
3. 给出新增或删除机制身份；
4. 重新运行2、3、4人正式测试；
5. 更新具体表达；
6. 不得直接覆盖当前基线记录。
