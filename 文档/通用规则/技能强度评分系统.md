# 通用技能强度评分系统

版本：GEN-SCORE-001  
生效日期：2026-07-14

## 一、文件定位

本文件是通用规则层的技能理论强度评分方法。

评分只评价规则效果本身，不评价：

- 具体名称；
- 世界观符合度；
- 插画表现；
- 收藏价值；
- AI在某一局面的使用优先级；
- 某一次具体战斗方案的收益。

技能强度评分不能替代模拟测试和实体试玩。

## 二、核心原则

```text
整数评分
无上限累计分
锚点优先
同一收益不得重复计算
```

推荐公式：

```text
技能强度评分
= 抽象机制锚点分
+ 超出锚点的新收益修正
- 限制与成本修正
```

完整判断维度：

```text
作用对象
× 动作类型
× 受影响战力
× 影响数量
× 持续时间
× 牌区去向
× 区域连锁
× 行动节奏
× 资源差
× 选择自由度
× 多玩家或多区域范围
- 等级限制
- 前置条件
- 使用成本
- 反制风险
```

## 三、强度区间

| 分数 | 层级 | 通用解释 |
|---:|---|---|
| 0 | 无效 | 没有实际信息、资源、场面或节奏收益 |
| 1–19 | 极弱 | 只有极轻微信息或提示作用 |
| 20–39 | 弱 | 有轻微收益，但限制明显 |
| 40–59 | 中 | 有明确场面、资源或节奏收益 |
| 60–79 | 强 | 能影响关键单位、区域或行动节奏 |
| 80–99 | 极强 | 大范围影响、强反制或接近胜负关键 |
| 100–149 | 超强 | 能显著重置单个关键区域或形成高资源差 |
| 150–219 | 破局级 | 同时强影响多个区域、牌区或玩家 |
| 220以上 | 失控级 | 多个核心系统同时被重置或夺取，通常不适合基础牌库 |

100是“返回一个基础容量区域内全部单位并触发区域回流”的通用参照点，不是满分。

## 四、锚点优先原则

评分时先选择最接近的抽象机制锚点。

若锚点已经包含：

- 对象数量；
- 受影响战力；
- 区域变为无主；
- 区域回流；
- 重新部署所需的行动节奏；

则不得再次把这些内容重复加分。

只对锚点之外的新效果继续修正。

## 六、受影响战力修正

应同时记录：

```text
理论最高影响战力
常见局面影响战力
```

| 受影响战力 | 建议修正 |
|---:|---:|
| 1 | +0 |
| 2–3 | +5 |
| 4–6 | +10 |
| 7–9 | +20 |
| 10–12 | +30 |
| 13–18 | +45 |
| 19以上 | +60起，按实际战力继续累计 |

若锚点已包含该战力区间，不重复加分；只有超过锚点包含的战力部分才继续修正。

## 八、资源差与牌区去向

同样影响相同战力时，去向强度通常为：

```text
返回拥有者手牌
< 进入弃牌堆
< 加入使用者手牌
< 加入使用者控制区域
```

参考修正：

| 去向或资源效果 | 修正 |
|---|---:|
| 返回拥有者手牌 | +0 |
| 进入弃牌堆 | +20 |
| 加入使用者手牌 | +50 |
| 加入使用者区域 | +60 |
| 产生明确双向资源差 | +10 |
| 获得另一玩家1张手牌 | +20 |
| 获得一名玩家全部手牌 | +80起 |
| 获得其他所有玩家全部手牌 | +160起 |
| 一个区域全部单位进入使用者手牌 | +50 |
| 全场单位进入使用者手牌 | +120起 |

如所选锚点已直接定义该去向，不重复加分。

## 九、行动节奏修正

| 行动影响 | 修正 |
|---|---:|
| 不影响行动 | 0 |
| 额外查看或轻微动作 | +5 |
| 额外部署1个单位 | +12 |
| 额外进行1次进攻 | +18 |
| 额外使用1张效果牌 | +22 |
| 额外执行1个完整主要行动 | +28 |

## 十、持续时间修正

| 持续时间 | 修正 |
|---|---:|
| 立即一次性结算 | 0 |
| 持续到本回合结束 | +5 |
| 持续到使用者下回合开始 | +7 |
| 持续到目标区域失控 | +12 |
| 永久持续 | +18，基础版慎用 |

## 十一、区域连锁修正

通用区域连锁：

```text
单位被移走或击败
→ 区域失去全部单位
→ 区域变为无主
→ 区域回流
→ 中央区域补充
```

| 连锁能力 | 修正 |
|---|---:|
| 不可能触发区域无主 | 0 |
| 可能触发区域无主 | +5 |
| 经常触发区域无主 | +10 |
| 必定触发1个区域无主 | +14 |
| 必定触发多个区域无主 | +30起 |
| 同时触发1个区域回流与中央补充 | +20 |
| 同时触发多个区域回流与中央补充 | +45起 |
| 直接夺取1个区域 | +20，基础版慎用 |
| 直接夺取多个区域 | +50起，基础版慎用 |

## 十二、限制与成本修正

| 限制或成本 | 修正 |
|---|---:|
| 只能影响1级单位 | -10 |
| 只能影响2级以下单位 | -4 |
| 只能选择己方区域 | -6 |
| 只能选择自己 | -12 |
| 需要指定机制身份在场 | -14 |
| 需要指定分组或指定区域 | -14 |
| 只在对手即将获胜时可用 | -20 |
| 占用主要行动 | -6 |
| 一次性效果牌 | -4 |
| 可被取消效果反制 | -4 |
| 合法目标容易不存在 | -6 |
| 目标返回手牌并保留资源 | -4 |
| 效果公开且容易规避 | -6 |

正式候选技能扣分后最低记为1；理论无效锚点可以记为0。

## 十三、胜利相关性

胜利相关性通常是临场修正，不写入技能基础分。

| 局面 | 临场修正 |
|---|---:|
| 普通区域 | 0 |
| 对手已控制同组2个区域 | +10 |
| 使用者已控制同组2个区域 | +10 |
| 能阻止对手立即胜利 | +12 |
| 能让使用者立即胜利 | +20，并单独测试 |

## 十四、输出格式

```text
机制效果：<抽象效果>
锚点分：<整数>
超出锚点修正：<逐项列出>
限制与成本修正：<逐项列出>
技能强度评分：<整数>
理论最高影响战力：<数值或范围>
常见影响战力：<数值或范围>
评分理由：<简要说明>
测试要求：<需要重点验证的风险>
```

## 十五、与其他评分系统的边界

```text
技能强度评分
≠ AI卡牌优先级评分
≠ 战斗方案评分
```

本文件给出的分数不能直接作为AI决策权重，也不能直接作为某次进攻的收益值。
