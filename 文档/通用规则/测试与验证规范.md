# 测试与验证规范

规则版本：V1.3  
更新时间：2026-07-14

## 一、测试层级

### 1. 单元测试

验证：

- 同时伤害；
- 防守伤害累计；
- 双方全灭；
- 区域回流；
- 中央区域补充；
- 驻军上限；
- 即时胜利；
- 牌库耗尽比较；
- 非法行动阻止。

### 2. 状态机回归

验证完整流程：

```text
开局
→ 回合
→ 部署
→ 进攻
→ 效果
→ 回流
→ 即时胜利
→ 牌库耗尽
```

### 3. 蒙特卡洛模拟

使用固定配置批量运行，比较：

- 2人、3人、4人；
- 基础牌库与扩展牌库；
- 双人6、9、12区域；
- human_like与stress_attack。

### 4. 实体试玩

重点验证AI无法覆盖的内容：

- 社交回应；
- 欺骗与读心；
- 规则理解；
- 挫败感；
- 决策时间；
- 卡面可读性；
- 桌面空间；
- 互动是否有趣。


## 五、通过条件

不设置单一绝对阈值。必须比较：

- 上一稳定版本；
- 当前候选版本；
- 不同人数；
- 不同区域参数；
- 基础与扩展牌库。

出现以下情况必须人工复核：

- 超时率明显升高；
- P90回合大幅增加；
- 直接胜利率显著下降；
- 区域回流异常增加；
- 双方全灭或回流造成循环；
- 某玩家位置胜率明显偏离；
- 测试结果无法用相同种子复现。

## 六、代码要求

- Python 3.12。
- 不依赖具体卡名。
- 使用机制ID和效果ID。
- 核心规则与AI策略分离。
- human_like与stress_attack分离。
- 测试输出支持JSON。
- 固定种子可复现。
- 代码通过 `python -m unittest`。

## 七、CI建议

每次修改通用规则或引擎时：

1. 编译所有Python文件。
2. 运行单元测试。
3. 运行2、3、4人烟雾模拟。
4. 检查区域牌库与公共牌库数量。
5. 保存JSON结果为构建产物。

正式大样本测试不必阻塞每次小提交，可在规则候选锁定时运行。
