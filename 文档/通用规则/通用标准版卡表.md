## 一、文件定位

本文件是 `data/core_set.json` 的人类可读说明。

标准版机制身份、数量、等级、类型、效果ID、参数和区域结构的唯一数据来源是：

```text
data/core_set.json
```

本文件不包含具体卡名、角色、世界观、语言文案、插画、具体表达卡号或包装内容。若与JSON冲突，以JSON为准并修正本文件。

## 三、字段说明

| 字段 | 含义 |
|---|---|
| `mechanic_id` | 不随具体表达变化的机制身份 |
| `kind` | `unit` 或 `effect` |
| `level` | 单位等级；效果牌为空 |
| `copies` | 标准版实体数量 |
| `effect_id` | 通用效果类型 |
| `parameters` | 效果数量、等级限制和目标范围 |
| `timing` | 触发或使用时机 |

## 十一、执行边界

- `CORE-U09` 查看牌库顶2张，选择1张抽取，未选牌放牌库底；牌库只剩1张时直接抽取该牌。
- `CORE-U11` 只能选择有手牌的其他玩家；双方必须暗选并同时交换，交换后改变两张牌的拥有者。
- `exact_count=true` 时，合法目标不足规定数量，效果不发动。
- `ownership_changes=true` 时，卡牌拥有者改为效果使用者。
- 返回手牌不视为击败。
- 任何移除最后驻军的效果立即触发区域回流。
- `PLAY_EXTRA_UNIT` 产生的单位正常触发入场效果。
- `IMMEDIATE_ATTACK` 完整遵守普通进攻流程。
- `CANCEL_EFFECT` 不占响应者自己回合的主要行动，不形成复杂反制连锁；取消后双方效果牌进入弃牌堆。
- `PROTECT_AREA` 为持续放置效果，保护使用者控制的1个区域及其中驻军，免疫其他玩家的一切进攻和效果；使用者自己的效果仍可影响该区域。
- `PROTECT_AREA` 持续到被取消或该区域变为无主，结束后进入弃牌堆。
