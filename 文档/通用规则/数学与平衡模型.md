# 数学与平衡模型

规则版本：V1.3  
更新时间：2026-07-14

## 一、文件定位

本文件只整理当前规则和现有测试中已经使用的通用数学关系与统计指标，不新增评价指数或校准公式。

数学模型不替代真人试玩。

## 二、战斗数学

单位等级为 `s`：

```text
health = s
damage = s
```

一次对决中，等级为 `a` 的进攻单位与等级为 `d` 的防守单位同时造成伤害。

防守单位在连续对决中的累计伤害：

```text
defender_damage_total += attacker_level
```

进攻单位每次战斗只攻击一次，受到的反击伤害不转移给其他进攻单位。

## 三、区域胜利

设玩家 `p` 在分组 `g` 中控制区域数量为：

```text
C(p, g)
```

即时胜利条件：

```text
存在 g，使 C(p, g) = 3
```

该条件与玩家控制的区域总数无直接等价关系，因此测试必须同时记录总控制区域和各分组控制情况。

## 四、牌库压力

设公共牌库数量为 `D`，玩家人数为 `P`，起始手牌为 `H0`。

开局后剩余牌数：

```text
D_remaining = D - P × H0
```

忽略额外抽牌时，基础抽牌轮数近似：

```text
R_base = floor(D_remaining / P)
```

实际耗尽速度还受以下当前机制影响：

- 额外抽牌；
- 查看牌库后抽取；
- 卡牌洗回牌库；
- 扩展增加牌库容量；
- 手牌交换和获得其他玩家手牌不直接减少公共牌库。

## 六、当前AI测试模型

### human_like

用于正式规则与参数测试，当前源行为包括：

- 优先接近胜利的区域；
- 保留基本防守；
- 不执行低于当前阈值的进攻；
- 按当前AI优先级选择卡牌；
- 社交回应采用当前概率；
- 不使用未公开隐藏信息。

### pressure

用于放大进攻和区域回流风险：

- 进攻执行阈值低于 `human_like`；
- 社交回应概率高于 `human_like`；
- 用于暴露超时、震荡和状态错误；
- 不作为正常平衡结论的唯一依据。

## 八、评价边界

- 直接胜利率高不代表体验一定好。
- 平均回合低不代表策略一定充分。
- 牌库耗尽率需要结合局时和结算情况解释。
- 区域回流次数只用于描述互动与震荡程度。
- AI结果用于规则风险筛查和参数对比，不替代实体试玩。
