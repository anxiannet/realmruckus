# 数学与平衡模型

规则版本：V1.3  
更新时间：2026-07-14

## 一、模型目的

数学模型用于：

- 检查规则是否能稳定收束；
- 比较不同人数与区域参数；
- 比较基础牌库与扩展牌库；
- 识别极端震荡、牌库耗尽和长尾风险；
- 为实体试玩提供重点观察项。

数学模型不替代真人试玩。

## 二、战斗模型

单位等级为 `s`：

```text
health = s
damage = s
```

一次对决中，等级为 `a` 的进攻单位与等级为 `d` 的防守单位同时造成伤害：

```text
attacker_remaining = a - d
defender_remaining = d - a
```

但防守单位在连续对决中的伤害需要累计，因此实际状态为：

```text
defender_damage_total += attacker_level
```

进攻单位每次战斗只攻击一次，受到的反击不转移给其他进攻单位。

## 三、区域胜利模型

设玩家 `p` 在分组 `g` 中控制区域数量为：

```text
C(p, g)
```

即时胜利条件：

```text
存在 g，使 C(p, g) = 3
```

该条件与总控制区域数无关，因此模型必须同时记录：

- 总区域数；
- 各分组区域数；
- 距离同组3区的差值。

推荐进度值：

```text
need(p, g) = 3 - C(p, g)
```

## 四、牌库压力

设公共牌库数量为 `D`，玩家人数为 `P`，起始手牌为 `H0`。

开局后剩余牌数：

```text
D_remaining = D - P × H0
```

忽略额外抽牌时，理论基础抽牌轮数近似：

```text
R_base = floor(D_remaining / P)
```

实际牌库耗尽受以下因素影响：

- 额外抽牌；
- 查看牌库后抽取；
- 卡牌洗回牌库；
- 手牌交换和偷取不改变牌库数量；
- 扩展增加的牌库容量。

## 五、进度与震荡指标

每局至少记录：

```text
turns
winner_type
controlled_areas
successful_attacks
failed_attacks
area_returns
deck_exhausted
settlement_triggered
timeout
```

推荐衍生指标：

```text
direct_win_rate
settlement_rate
draw_rate
timeout_rate
average_turns
median_turns
p90_turns
deck_exhaustion_rate
average_successful_attacks
average_area_returns
```

### 震荡指数

可用以下简化值观察区域反复易手：

```text
oscillation_index = area_returns / max(1, successful_attacks)
```

该值只用于版本间相对比较，不设绝对合格线。

### 收束效率

```text
closure_efficiency = direct_wins / total_games
```

需要结合平均回合、P90和超时率解释，不能单独作为平衡结论。

## 六、AI模型分层

### human_like

正式平衡模型。目标是近似普通玩家：

- 优先接近胜利的区域；
- 保留合理防守；
- 不进行无意义攻击；
- 根据手牌和局势使用效果；
- 社交回应采用概率模型；
- 不掌握隐藏信息。

### stress_attack

压力模型。目标是放大高频进攻与区域回流：

- 更高进攻倾向；
- 更少保留驻军；
- 用于发现无限震荡、超时和状态错误；
- 结果不得用于正常平衡结论。

## 七、随机与复现

所有批量测试必须记录：

```text
rules_version
deck_version
model_version
player_count
area_group_count
center_size
games
seed
max_turns
```

同一配置使用固定随机种子以支持回归比较。

## 八、评价原则

- 直接胜利率高，不代表体验一定好。
- 平均回合低，不代表策略足够。
- 牌库耗尽率高，需要检查是否为有意义的终局压力。
- 区域回流高，需要检查是互动丰富还是无效震荡。
- 扩展牌库应同时比较局时、耗尽率、超时率和关键效果触发。
- AI结果只用于风险筛查、回归和参数比较。
